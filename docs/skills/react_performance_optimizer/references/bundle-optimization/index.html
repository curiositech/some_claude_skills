Bundle Optimization Strategies

Comprehensive guide to reducing JavaScript bundle size for faster load times.

Why Bundle Size Mattersâ€‹

Impact on User Experience:

  • 100KB bundle (gzipped) â†’ ~1s load on 3G
  • 500KB bundle (gzipped) â†’ ~5s load on 3G
  • Every 100KB adds ~1s to Time to Interactive

Business Impact:

  • Pinterest: -40% load time â†’ +15% conversions
  • AutoAnything: -50% load time â†’ +12-13% sales
  • BBC: Every 1s slower â†’ -10% users

Strategy 1: Code Splittingâ€‹

Route-Based Splitting (Easiest Win)â€‹

Before (single bundle):

import Home from './pages/Home';
import About from './pages/About';
import Dashboard from './pages/Dashboard';  // 500KB

After (split by route):

import { lazy, Suspense } from 'react';

const Home = lazy(() => import('./pages/Home'));
const About = lazy(() => import('./pages/About'));
const Dashboard = lazy(() => import('./pages/Dashboard'));

function App() {
  return (
    <Suspense fallback={<Loading />}>
      <Routes>
        <Route path="/" element={<Home />} />
        <Route path="/about" element={<About />} />
        <Route path="/dashboard" element={<Dashboard />} />
      </Routes>
    </Suspense>
  );
}

Impact: Main bundle: 800KB â†’ 300KB (500KB lazy-loaded)

Component-Based Splittingâ€‹

Split heavy components that aren't always shown:

import { lazy, Suspense } from 'react';

// âŒ Always loaded (even if modal never opens)
import PDFViewer from './PDFViewer';  // 200KB

// âœ… Loaded on demand
const PDFViewer = lazy(() => import('./PDFViewer'));

function App() {
  const [showPDF, setShowPDF] = useState(false);

  return (
    <div>
      <button onClick={() => setShowPDF(true)}>View PDF</button>

      {showPDF && (
        <Suspense fallback={<Spinner />}>
          <PDFViewer url="/document.pdf" />
        </Suspense>
      )}
    </div>
  );
}

When to split:

  • Modals/dialogs
  • Admin panels
  • Charts/visualizations
  • Rich text editors
  • Video players

Vendor Splittingâ€‹

Separate third-party code from your app code:

webpack.config.js:

module.exports = {
  optimization: {
    splitChunks: {
      cacheGroups: {
        // Vendor chunk (rarely changes)
        vendor: {
          test: /[\\/]node_modules[\\/]/,
          name: 'vendors',
          chunks: 'all',
        },
        // Common code shared across pages
        common: {
          minChunks: 2,
          priority: -10,
          reuseExistingChunk: true,
        },
      },
    },
  },
};

Benefit: Vendor bundle cached long-term (changes infrequently)

Strategy 2: Tree Shakingâ€‹

Use ES Modules (ESM)â€‹

âŒ CommonJS (entire library imported):

const _ = require('lodash');  // 500KB
_.debounce(() => {}, 300);

âœ… ES Modules (only imported function included):

import { debounce } from 'lodash-es';  // ~5KB
debounce(() => {}, 300);

Impact: 500KB â†’ 5KB (99% reduction)

Import Only What You Needâ€‹

âŒ Whole library:

import * as MUI from '@mui/material';  // 300KB

function App() {
  return <MUI.Button>Click</MUI.Button>;
}

âœ… Specific imports:

import Button from '@mui/material/Button';  // 50KB

function App() {
  return <Button>Click</Button>;
}

Configure Babel for Tree Shakingâ€‹

.babelrc:

{
  "presets": [
    ["@babel/preset-env", {
      "modules": false  // Don't transform ES modules
    }]
  ]
}

Without "modules": false, Babel converts ESM to CommonJS, breaking tree-shaking.

Strategy 3: Replace Heavy Dependenciesâ€‹

Common Swapsâ€‹

Old LibrarySizeNew LibrarySizeSavings
moment.js288KBdate-fns28KB90%
lodash500KBlodash-es5KB*99%*
axios50KBnative fetch0KB100%
react-router-dom45KBwouter1.2KB97%

* When importing only needed functions

Example: Replace moment.js with date-fnsâ€‹

Before:

import moment from 'moment';  // 288KB

const formatted = moment(date).format('MMM DD, YYYY');
const relative = moment(date).fromNow();

After:

import { format, formatDistanceToNow } from 'date-fns';  // 28KB

const formatted = format(date, 'MMM dd, yyyy');
const relative = formatDistanceToNow(date, { addSuffix: true });

Impact: -260KB (-90%)

Example: Replace axios with fetchâ€‹

Before:

import axios from 'axios';  // 50KB

const response = await axios.get('/api/users');
const data = response.data;

After:

// Native fetch (0KB)
const response = await fetch('/api/users');
const data = await response.json();

Impact: -50KB (-100%)

Note: For complex use cases, consider ky (5KB) as lightweight axios alternative

Strategy 4: Externalize Large Dependenciesâ€‹

Move rarely-changing libraries to CDN:

index.html:

<!-- React from CDN -->
<script crossorigin src="https://unpkg.com/react@18/umd/react.production.min.js"></script>
<script crossorigin src="https://unpkg.com/react-dom@18/umd/react-dom.production.min.js"></script>

webpack.config.js:

module.exports = {
  externals: {
    react: 'React',
    'react-dom': 'ReactDOM',
  },
};

Impact: Bundle size: 500KB â†’ 450KB (React now loaded from CDN)

Trade-offs:

  • âœ… Smaller bundle
  • âœ… CDN caching across sites
  • âŒ Extra HTTP request
  • âŒ Dependency on CDN availability

Strategy 5: Remove Unused Codeâ€‹

Analyze with Bundle Analyzerâ€‹

npx webpack-bundle-analyzer stats.json

Look for:

  • Libraries you don't remember installing
  • Multiple versions of same library
  • Test utilities in production bundle
  • Polyfills for features you don't use

Example: Remove Unused Polyfillsâ€‹

Before (polyfills for IE11):

import 'core-js/stable';  // 200KB
import 'regenerator-runtime/runtime';  // 50KB

After (modern browsers only):

// Remove polyfills
// Assume target: "es2020" in tsconfig.json

Impact: -250KB if targeting modern browsers

Remove Development-Only Codeâ€‹

.babelrc (production):

{
  "plugins": [
    ["transform-remove-console", { "exclude": ["error", "warn"] }]
  ]
}

webpack.config.js:

module.exports = {
  plugins: [
    new webpack.DefinePlugin({
      'process.env.NODE_ENV': JSON.stringify('production')
    })
  ]
};

This removes PropTypes, debug logs, and dev warnings.

Lazy Load Imagesâ€‹

function ImageGallery({ images }) {
  return (
    <div>
      {images.map(img => (
        <img
          key={img.id}
          src={img.url}
          loading="lazy"  // Native lazy loading
          alt={img.alt}
        />
      ))}
    </div>
  );
}

Impact: Initial page load doesn't download off-screen images

Inline Small Assetsâ€‹

webpack.config.js:

module.exports = {
  module: {
    rules: [
      {
        test: /\.(png|jpg|gif|svg)$/,
        type: 'asset',
        parser: {
          dataUrlCondition: {
            maxSize: 8 * 1024  // Inline if &lt;8KB
          }
        }
      }
    ]
  }
};

Benefit: Small images become base64 data URLs (no extra HTTP requests)

Strategy 7: Compressionâ€‹

Enable Gzip/Brotliâ€‹

Server config (nginx):

# Gzip
gzip on;
gzip_types text/plain text/css application/json application/javascript;
gzip_min_length 1000;

# Brotli (better compression)
brotli on;
brotli_types text/plain text/css application/json application/javascript;

Impact: 500KB bundle â†’ 100KB gzipped (80% reduction)

Brotli vs Gzip: Brotli ~20% smaller than Gzip for text

Pre-compress at Build Timeâ€‹

# Generate .gz and .br files at build
npx webpack --mode production
gzip -k dist/*.js
brotli dist/*.js

Benefit: Server doesn't compress on-the-fly (faster response)

Strategy 8: Minificationâ€‹

Terser (Webpack Default)â€‹

webpack.config.js:

module.exports = {

Strategy 9: Dynamic Imports for Featuresâ€‹

Split features that not all users need:

function App() {
  const [showAdmin, setShowAdmin] = useState(false);

  const loadAdminPanel = async () => {
    // Only load admin code when needed
    const { AdminPanel } = await import('./AdminPanel');
    setShowAdmin(true);
  };

  return (
    <div>
      {user.isAdmin && (
        <button onClick={loadAdminPanel}>Open Admin</button>
      )}

      {showAdmin && <AdminPanel />}
    </div>
  );
}

Impact: Admin code (200KB) not loaded for regular users

Strategy 10: Prefetch/Preload Strategic Resourcesâ€‹

Prefetch Next Pageâ€‹

import { lazy, Suspense } from 'react';

const Dashboard = lazy(() => import('./Dashboard'));

function Home() {
  useEffect(() => {
    // Prefetch dashboard during idle time
    const link = document.createElement('link');
    link.rel = 'prefetch';
    link.href = '/dashboard.js';
    document.head.appendChild(link);
  }, []);

  return <div>Home</div>;
}

Benefit: Dashboard loads instantly when user navigates to it

Preload Critical Resourcesâ€‹

<head>
  <!-- Load critical font before anything else -->
  <link rel="preload" href="/fonts/main.woff2" as="font" type="font/woff2" crossorigin>

  <!-- Preload critical CSS -->
  <link rel="preload" href="/critical.css" as="style">
</head>

Real-World Exampleâ€‹

Problem: 2.5MB Bundleâ€‹

Before:

Main bundle:     2.5MB (uncompressed)
Gzipped:         650KB
Time to Interactive: 8 seconds on 3G

Issues Found:

  1. Entire app in one bundle (no code splitting)
  2. moment.js (288KB) for simple date formatting
  3. Lodash CommonJS import (500KB)
  4. Chart library always loaded (400KB)
  5. Multiple polyfills (250KB)
  6. No compression

Fixes Applied:

  1. Route-based code splitting â†’ -800KB
  2. Replace moment.js with date-fns â†’ -260KB
  3. Use lodash-es with tree shaking â†’ -495KB
  4. Lazy load chart component â†’ -400KB
  5. Remove unnecessary polyfills â†’ -250KB
  6. Enable Brotli compression â†’ 80% reduction

After:

Main bundle:     250KB (uncompressed)
Brotli:          50KB
Time to Interactive: 1.2 seconds on 3G

Result: 8s â†’ 1.2s (85% faster)
