What is this? A Directed Acyclic Graph (DAG) showing task execution order and dependencies.

How Data Flows:

  • Nodes (Windows): Each window represents a skill/agent that performs work
  • Arrows: Show dependencies - an arrow from Aâ†’B means "B depends on A's output"
  • Waves (Columns): Nodes in the same column can run in parallel (no dependencies between them)
  • Execution Order: Left to right - earlier waves complete before later ones

Node Inputs & Outputs:

  • Input: Each node receives the combined outputs of all its dependencies
  • Processing: The skill analyzes inputs and produces new artifacts (docs, designs, code, etc.)
  • Output: Artifacts are passed to dependent nodes (shown by arrows)
  • Example: "Research" outputs findings â†’ "Write" consumes findings and outputs draft â†’ "Review" consumes draft and outputs feedback

Color Legend:

  • Navy: Pending (waiting for dependencies)
  • Yellow: Ready (all dependencies met, can start)
  • Blue: Running (actively executing)
  • Green: Completed (output available)
  • Red: Failed (error occurred)
